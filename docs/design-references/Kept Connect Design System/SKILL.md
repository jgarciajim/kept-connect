If creating visual artifacts (slides, mocks, throwaway prototypes, etc), copy assets out and create static HTML files for the user to view. If working on production code, you can copy assets and read the rules here to become an expert in designing with this brand.

If the user invokes this skill without any other guidance, ask them what they want to build or design, ask some questions, and act as an expert designer who outputs HTML artifacts _or_ production code, depending on the need.

## Quick orientation

- **`readme.md`** — the full design guide: north star, content fundamentals (voice), visual foundations, iconography, the mark, and a file index. Read it first.
- **`styles.css`** — link this one file to inherit every token and font. `@import`s `tokens/*`.
- **`tokens/`** — `colors.css`, `typography.css`, `spacing.css`, `fonts.css`. All CSS custom properties.
- **`components/`** — React primitives: `KeptConnectLogo`, `Button`, `Card`, `CategoryIcon`, `Input`, `Avatar`, `Tag`, `VerifiedBadge`. Each has a `.prompt.md` with usage.
- **`guidelines/`** — foundation specimen cards (colors, type, spacing, brand).
- **`ui_kits/requester/`** — the warm consumer app (home → post → live match → quotes → track).
- **`ui_kits/provider/`** — the cool dark trades app (job feed, live offers, fast pay, active job).
- **`assets/`** — the six `logo-*.svg` mark treatments.

## Using this in Claude Code

- **For production work:** the files under `components/<group>/<Name>.jsx` are real, importable React modules (`import React from 'react'; export function <Name>(…)`). Read each component's `.jsx` + `.d.ts` (props) + `.prompt.md` (usage) and recreate or port it into the target codebase's framework and patterns. Consume design values via the CSS custom properties in `styles.css` / `tokens/*` — never hard-code hexes.
- **For throwaway HTML/mocks:** the prebuilt `_ds_bundle.js` at the root exposes every component on `window.KeptConnectDesignSystem_4a1eb7`. Link `styles.css`, add React + ReactDOM + Babel, `<script src>` the bundle, then `const { Button, CategoryIcon } = window.KeptConnectDesignSystem_4a1eb7`. The `ui_kits/*/index.html` files are working examples of exactly this.
- **The bundle (`_ds_bundle.js`) and `_ds_manifest.json` are prebuilt artifacts** generated by the authoring environment. Outside it they do **not** auto-regenerate — if you edit a component's `.jsx`, update consumers from the source directly (the bundle won't refresh on its own).
- **Assets:** copy the SVGs you need out of `assets/`; don't hot-link across folders.

## The rules to never break

1. **The warmth budget.** Ask: is this surface *welcoming* someone or *informing* them? Welcoming → `--moment` cream, Fraunces, air. Informing (any numbers) → clean `--paper`, tabular, near-mono. Never warm-wash data.
2. **Color carries meaning — three tiers, nothing decorative.** **Terracotta** = brand + every action + live (one terracotta primary per screen, never two). **Emerald** = verified, and only verified. **Category hues** = which trade, only on trade icons (neutral chip + colored glyph). Everything else is black-and-white. On dark, use the `-bright` terracotta/verified variants.

Sentence case everywhere. Weights 400/500 only. The signature **terracotta period** ends key headlines (`Booked.` `On the way.`). No emoji. Trust is rendered as calm, not badges.
